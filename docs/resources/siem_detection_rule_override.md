# rapid7_siem_detection_rule_override (Resource)

Manages an [override applied to a detection rule](https://docs.rapid7.com/insightidr/modify-detection-rules) At least one of `action` or `priority` must be specified



## Example

```terraform
resource "rapid7_siem_detection_rule_override" "rules_0" {
  rule_rrn = "rrn:cba:::detection-rule:0GYFI91AVC4K"

  action   = "CREATES_ALERTS"
  priority = "HIGH"
}
```

## Schema

### Required

- `rule_rrn` (String) Identifier of the rule which this override is applied to

### Optional

- `action` (String) Action taken when the detection rule matches an event
  * `OFF` - No detection is generated. Priority is not applicable.
  * `ASSESS_ACTIVITY` - Statistics are collected, but no detection is generated. Priority is not applicable.
  * `TRACKS_NOTABLE_EVENTS` - A notable event is added to an open investigation, if one exists. Priority is not applicable.
  * `CREATES_ALERTS` - An alert is created. Priority is required to be one of `INFO` `LOW` `MEDIUM` `HIGH` `CRITICAL`
  * `CREATES_INVESTIGATIONS` - An investigation is opened. Priority is required to be one of `LOW` `MEDIUM` `HIGH` `CRITICAL`
- `priority` (String) Priority associated with the action, required when applicable
  * `INFO` - Informational
  * `LOW` - Low
  * `MEDIUM` - Medium
  * `HIGH` - High
  * `CRITICAL` - Critical
